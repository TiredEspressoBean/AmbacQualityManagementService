## Traceability Implementation

### Part Traceability

```
Order
  └── WorkOrder
        └── Parts
              ├── serial_number / ERP_id
              ├── StepTransitionLog (what/when/who)
              ├── QualityReports (inspections)
              ├── MeasurementResults (data)
              └── Documents (attachments)
```

### Audit Trail

| Event | Tracked Data |
|-------|--------------|
| Record changes | Before/after values, user, timestamp |
| Step transitions | Part, step, operator, machine, timestamp |
| Approvals | Approver, timestamp, signature, IP address |
| Permission changes | Who granted/revoked, to whom, when |

## Document Control Details

### Document Status Workflow

```
DRAFT ──────▶ UNDER_REVIEW ──────▶ APPROVED ──────▶ RELEASED
                   │                                    │
                   ▼                                    ▼
               (Rejected)                           OBSOLETE
```

### Classification Levels

| Level | Access |
|-------|--------|
| PUBLIC | All users |
| INTERNAL | Employees |
| CONFIDENTIAL | Managers+ |
| RESTRICTED | Limited access |
| SECRET | Critical access |
