## 8D Problem Solving Coverage

The CAPA system implements all 8 disciplines:

| Discipline | Implementation |
|------------|----------------|
| D1: Team | `assigned_to` field, team member assignments |
| D2: Problem | `problem_statement` field |
| D3: Containment | `immediate_action` field; Containment task type |
| D4: Root Cause | RcaRecord with 5 Whys, Fishbone (6M categories) |
| D5: Corrective Actions | CapaTasks with CORRECTIVE type |
| D6: Implementation | Task completion tracking with dates |
| D7: Preventive Actions | CapaTasks with PREVENTIVE type |
| D8: Verification | CapaVerification with `effectiveness_result` |

**Key files**: `Tracker/models/qms.py` (CAPA, CapaTasks, RcaRecord, CapaVerification)

- SPC monitoring requirements
- Enhanced control plan visibility

## Rework Control

Per clause 8.7.1.4, rework is controlled:

| Feature | Implementation |
|---------|----------------|
| Visit counting | `Part.rework_attempt_at_step` increments per revisit |
| Maximum visits | `Steps.max_visits` limits rework attempts |
| Rework routing | Disposition workflow routes to rework step |
| Re-inspection | Required after rework completion |
