# PDF Export Requirements - Auto/Aero QMS

**Last Updated:** April 2026

This document tracks all PDF documents that the QMS software should
generate for automotive (IATF 16949) and aerospace (AS9100D) compliance.

**Scope:** Software-generated PDFs only. Documents created externally
(CAD drawings, FMEAs in Excel, etc.) are stored via the document
management system but not generated by this software.

## PDF Generation Implementation Details

### ✅ Implemented Technology Stack

**Typst** renders templates to PDF directly from a Python binding —
no headless browser, no frontend round-trip, no authentication gymnastics.
Each report type is an **adapter** (Python class) that validates
params, queries the ORM, and returns a Pydantic model which is
serialized to JSON and fed to a Typst template.

- **PDF Engine:** Typst via `typst` pip package (Rust backend, pre-built manylinux wheels)
- **Context Models:** Pydantic — typed contract between adapter and template
- **Template Engine:** Typst (markup-based typesetter, serif/sans/mono via `fonts-noto-core` apt package)
- **Community packages:** `cetz`, `cetz-plot`, `diagraph` — vendored in `PartsTracker/typst_packages/` (works offline / on-prem)
- **Storage:** `GeneratedReport` model → linked to `Documents` (DMS integration)
- **API:** `/api/reports/generate/` (async email) and `/api/reports/download/` (sync download with 30s timeout)
- **Processing:** Celery async task (`RetryableTypstTask`: 30s soft / 60s hard limit, retry only on transient errors), or synchronous download

### Performance

| Scenario | Latency |
|----------|---------|
| Simple document (CoC, NCR, 8D, traveler) | ~50-150ms |
| Chart-heavy document (SPC with CeTZ) | ~200-500ms |
| DAG-heavy document (process routing via diagraph) | ~200-500ms |
| Cold compile (first request after worker start) | +100-200ms one-time cost |
| Label sheet (30 labels per Avery page) | ~100-200ms |

Key characteristics vs. the former browser-based pipeline:
- **No warm-browser RAM overhead** — ~500 MB/process saved
- **No network dependency at compile time** — no frontend roundtrip, no Typst Universe download (packages vendored)
- **Deterministic output** — byte-identical PDF for same inputs; important for archival / regeneration parity
- **Lazy `Compiler` singleton** per worker process, initialized on first request (~100ms), reused for subsequent compiles

### Architecture Flow

**Documents:**
```
User clicks "Download" or "Email Report"
        ↓
POST /api/reports/download/ or /api/reports/generate/
        ↓
ReportViewSet validates envelope {report_type, params}
        ↓
PdfGenerator.generate()
  → registry.get_adapter(report_type)
  → adapter.param_serializer_class.validate()   (tenant-scoping check)
  → adapter.build_context(validated_params, user, tenant) → Pydantic model
  → context.model_dump(mode="json") → dict
  → typst_generator.generate_typst_pdf(template_path, dict) → PDF bytes
        ↓
Returns PDF directly (sync) or saves to DMS + emails (async Celery)
        ↓
GeneratedReport record for audit trail
```

**Labels** (future — not yet implemented in Typst):

Label reports will follow the same adapter pattern with template-level
page-size overrides (e.g., 2.625"×1" Avery cells, custom grid layouts).
The Typst `#set page(width:, height:)` directive replaces Playwright's
per-type page-size config.

#### Step 1: Define the Pydantic context model

The context model is the contract between the adapter and the template.
Put it in the same file as the adapter unless it grows large enough to
warrant its own module.

```python
# Tracker/reports/adapters/cert_of_conformance.py

from __future__ import annotations
from datetime import date
from typing import Optional

from pydantic import BaseModel, Field


class PartLine(BaseModel):
    serial: str
    part_number: str
    status: str


class CertOfConformanceContext(BaseModel):
    certificate_number: str
    issue_date: date
    customer_name: str
    customer_po_number: Optional[str] = None
    parts: list[PartLine] = Field(default_factory=list)
    tenant_name: str
```

#### Step 2: Define the DRF param serializer

Validates params on the way in AND confirms the requested resource
belongs to the user's tenant (defense in depth — the adapter's
`build_context()` should also filter by tenant explicitly).

```python
from rest_framework import serializers


class CertOfConformanceParamsSerializer(serializers.Serializer):
    """
    Accepts {"id": <uuid>} and verifies the Order belongs to the
    requesting user's tenant.
    """
    id = serializers.UUIDField()  # Orders.id is a UUID (SecureModel)

    def validate_id(self, value):
        from Tracker.models import Orders

        user = self.context.get("user") or (
            getattr(self.context.get("request"), "user", None)
        )
        if user is None:
            raise serializers.ValidationError("Authenticated user required.")

        tenant = getattr(user, "_current_tenant", None) or getattr(user, "tenant", None)
        if tenant is None:
            raise serializers.ValidationError("No tenant context on user.")

        # tenant-safe: explicit tenant filter
        if not Orders.objects.filter(id=value, tenant=tenant).exists():
            raise serializers.ValidationError(f"Order {value} not found.")
        return value
```

---

#### Step 3: Define the adapter

Subclass `ReportAdapter`; fill in the required class attributes;
implement `build_context()` which queries the ORM and returns a
Pydantic instance.

```python
from Tracker.reports.adapters.base import ReportAdapter


class CertOfConformanceAdapter(ReportAdapter):
    name = "cert_of_conformance"
    title = "Certificate of Conformance"
    template_path = "cert_of_conformance.typ"
    context_model_class = CertOfConformanceContext
    param_serializer_class = CertOfConformanceParamsSerializer

    def build_context(self, validated_params, user, tenant) -> CertOfConformanceContext:
        from Tracker.models import Orders

        # tenant-safe: explicit tenant filter (defense in depth)
        order = (
            Orders.objects.filter(tenant=tenant)
            .select_related("customer", "company")
            .prefetch_related("parts__part_type")
            .get(id=validated_params["id"])
        )
        return CertOfConformanceContext(
            certificate_number=f"CoC-{order.id.hex[:8].upper()}",
            issue_date=order.created_at.date(),
            customer_name=order.customer.name if order.customer else "",
            customer_po_number=order.customer_po_number,
            parts=[
                PartLine(
                    serial=p.ERP_id,
                    part_number=p.part_type.name,
                    status=p.status,
                )
                for p in order.parts.all()
            ],
            tenant_name=tenant.name,
        )

    def get_filename(self, validated_params) -> str:
        return f"coc_{validated_params['id']}.pdf"
```

#### Step 4: Write the Typst template

Place at `Tracker/reports/templates/<name>.typ`. Import `_common/` for
shared styling. Read the context at the top via `json.decode()`.

```typst
// Tracker/reports/templates/cert_of_conformance.typ

#import "_common/page-setup.typ": *

#let data = json.decode(sys.inputs.at("data"))

#show: page-setup.with(
  title: "Certificate of Conformance",
  doc-id: data.certificate_number,
  classification: "Controlled Document",
)

#align(center)[
  #text(size: 9pt, fill: muted, tracking: 2pt, font: sans-font)[
    CERTIFICATE OF CONFORMANCE
  ]
  #v(2pt)
  #text(size: 22pt, weight: "bold", font: sans-font)[#data.certificate_number]
  #v(-4pt)
  #text(size: 10pt, fill: muted)[#data.customer_name]
]

#v(10pt)

#grid(
  columns: (1fr, 1fr),
  column-gutter: 16pt,
  [*Issued*: #data.issue_date],
  [*Customer PO*: #if data.customer_po_number == none [—] else [#data.customer_po_number]],
)

= Parts

#table(
  columns: (auto, 1fr, auto),
  align: (center, left, center),
  [*Serial*], [*Part Number*], [*Status*],
  ..data.parts.map(p => (
    [#p.serial], [#p.part_number], [#p.status]
  )).flatten()
)

// ... signature block, conformance statement, etc. ...
```

**Key Typst gotchas** (see `typst_generator.py` docstring for the full list):

- Use `json.decode(sys.inputs.at("data"))` — NOT `json(...)`, which is
  the file-loader form and will try to interpret the JSON string as
  a filesystem path.
- Refer to optional fields via `if data.field == none [—] else [...]`
- Reserved words (`auto`, `none`, `true`, `false`, `let`, etc.) can't
  be used as dict keys without quoting them.
- `#` inside bold/italic needs escaping: `[*\#*]` or use `No.` instead.

```python
# DB-backed cross-tenant security test — see Tracker/reports/tests/test_ncr.py
# NcrAdapterCrossTenantTests for the pattern to copy.
```

The mixin gives you 4 free tests (fixture validates, template

test pattern is in `test_ncr.py::NcrAdapterCrossTenantTests`.

#### Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| `UnknownReportError` | `report_type` not in registry | Confirm the dotted path is in `REPORT_ADAPTERS` |
| `ReportParamError` with `{"id": ["NCR <uuid> not found."]}` | Cross-tenant id, or genuinely missing row | Confirm id belongs to the requesting user's tenant |
| `ValidationError` from Pydantic on `build_context()` | Adapter returning wrong types | Pydantic error message names the field; match the DTO's type |
| Typst compile fails with "dictionary does not contain key 'X'" | Template reads a field the adapter didn't produce | Add the field to the Pydantic model + adapter; re-run `export_context` |
| Typst compile fails with "failed to load file (os error 123)" | Using `json(...)` instead of `json.decode(...)` | Typst's `json()` is a file loader; use `json.decode()` for strings |
| Cross-tenant test fails (user_b can't render own report) | `IntegerField` on a UUID pk, or missing tenant filter | Use `UUIDField` for SecureModel primary keys |
| Sync download returns 504 | Compile exceeded 30s | Move to async `/generate/` or optimize the template/ORM queries |
| Cold compile is slow (~200ms) | First request after worker start | Expected — Compiler is lazy-initialized once per process; subsequent compiles are fast |

### PSW (Part Submission Warrant) - PPAP Element 18

**Purpose:** Cover sheet for PPAP submission package. Required for every PPAP submission.

**AIAG Standard Format - Required Sections:**

1. **Part Information**
   - Part name, part number, customer part number
   - Engineering change level, drawing number/date
   - Additional engineering changes (if any)
   - Safety/regulated item designation (Y/N)

2. **Organization Information**
   - Supplier name, code, address
   - Customer name/division
   - Buyer/engineer contact

3. **Submission Information**
   - Submission reason: Initial, Engineering Change, Tooling Transfer, Correction, Other
   - PPAP submission level (1-5)
   - Requested submission date

4. **18-Element Checklist**
   | # | Element | Submitted | N/A | Remarks |
   |---|---------|-----------|-----|---------|
   | 1 | Design Records | ☐ | ☐ | |
   | 2 | Engineering Change Documents | ☐ | ☐ | |
   | ... | ... | ... | ... | ... |
   | 18 | Part Submission Warrant | ☐ | ☐ | |

5. **Declaration**
   - Statement of conformance
   - Authorized signature, title, date, phone

6. **Customer Use Only**
   - Disposition: Approved / Interim Approval / Rejected
   - Customer signature, date
   - Tracking number

**Submission Levels (AIAG):**

| Level | What Customer Receives |
|-------|----------------------|
| 1 | PSW + limited supporting data |
| 2 | PSW + product samples + limited data |
| 3 | PSW + product samples + complete data (default) |
| 4 | PSW + other as defined by customer |
| 5 | PSW + product samples + complete data at supplier site |

**Data Model Required - `PPAPSubmission`:**

```python
class PPAPSubmission(SecureModel):
    """Tracks PPAP submissions to customers."""

    class SubmissionReason(models.TextChoices):
        INITIAL = 'initial', 'Initial Submission'
        ENGINEERING_CHANGE = 'engineering_change', 'Engineering Change'
        TOOLING_TRANSFER = 'tooling_transfer', 'Tooling Transfer/Replacement'
        CORRECTION = 'correction', 'Correction of Discrepancy'
        OTHER = 'other', 'Other'

    class SubmissionLevel(models.IntegerChoices):
        LEVEL_1 = 1, 'Level 1 - PSW only'
        LEVEL_2 = 2, 'Level 2 - PSW + samples + limited data'
        LEVEL_3 = 3, 'Level 3 - PSW + samples + complete data'
        LEVEL_4 = 4, 'Level 4 - Customer defined'
        LEVEL_5 = 5, 'Level 5 - Complete review at supplier'

    class CustomerDecision(models.TextChoices):
        PENDING = 'pending', 'Pending Review'
        APPROVED = 'approved', 'Approved'
        INTERIM = 'interim', 'Interim Approval'
        REJECTED = 'rejected', 'Rejected'

    # Part reference
    part_type = models.ForeignKey(PartType, on_delete=models.CASCADE)

    # Submission details
    submission_reason = models.CharField(max_length=30, choices=SubmissionReason.choices)
    submission_level = models.IntegerField(choices=SubmissionLevel.choices, default=3)
    engineering_change_level = models.CharField(max_length=50, blank=True)
    is_safety_item = models.BooleanField(default=False)

    # 18-element checklist (JSON: {element_num: {submitted: bool, na: bool, remarks: str}})
    element_checklist = models.JSONField(default=dict)

    # Customer info
    customer = models.ForeignKey(Companies, on_delete=models.CASCADE)
    customer_contact = models.CharField(max_length=100, blank=True)

    # Approval
    submitted_at = models.DateTimeField(null=True, blank=True)
    submitted_by = models.ForeignKey(User, on_delete=models.SET_NULL, null=True)
    customer_decision = models.CharField(max_length=20, choices=CustomerDecision.choices, default='pending')
    customer_decision_date = models.DateTimeField(null=True, blank=True)
    customer_notes = models.TextField(blank=True)

    # Link to generated PSW document
    psw_document = models.ForeignKey(Documents, on_delete=models.SET_NULL, null=True, blank=True)
```

**Gap:** Requires PPAPSubmission model + UI for checklist management + PDF template

**Effort:** 2-3 weeks (model + API + UI + PDF)
